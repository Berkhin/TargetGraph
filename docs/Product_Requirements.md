# Product Requirements Document (PRD): TargetGraph.io

> **Видение vs реализовано.** Это продуктовое видение; часть сценариев пока не
> реализована. Сверка с кодом (см. [../ARCHITECTURE.md](../ARCHITECTURE.md)):
> сорсинг — **Apify** (не headless-браузер); поиск контактов — **Hunter.io** (а не
> SMTP-движок, который существует автономно — [Email_Verification_Spec.md](./Email_Verification_Spec.md));
> фоновые задачи — **APScheduler** (не TaskIQ/Celery); PDF — на клиенте (jsPDF);
> **парсинг входящих писем (Pub/Sub) пока НЕ реализован** — Gmail работает только
> на отправку.

## 1. Концепция и Цели
Полностью автономная система для поиска, оценки и отклика на вакансии. Система работает в фоновом режиме на домашнем Linux-сервере в Docker-контейнерах. 

**Главная отличительная черта:** Отказ от автоматизированного заполнения сложных веб-форм на сайтах компаний в пользу генерации персонализированных писем и адаптированных резюме с последующей прямой отправкой контактному лицу (рекрутеру/нанимающему менеджеру).

## 2. Архитектурные Принципы
* **Infrastructure as Code (IaC):** Вся система разворачивается через Docker Compose.
* **Asynchronous Processing:** Скрапинг вынесен в фоновую задачу планировщика
  (**APScheduler**); LLM-генерация и отправка писем выполняются асинхронно в рамках
  запроса (REST/WebSocket). Внешняя очередь задач (Celery/TaskIQ) не используется.
* **Rate Limiting & Human Mimicry:** Встроенные задержки и проксирование.

## 3. Пользовательские Сценарии (Use Cases)

### UC-1: Автоматический поиск вакансий (Auto-Sourcing)
Система по расписанию запускаетheadless-браузер или обращается к API для поиска релевантных позиций по заданным фильтрам. Новые вакансии сохраняются в базу со статусом `NEW`.

### UC-2: Оценка релевантности (AI Matching)
Фоновый воркер передает описание новой вакансии вместе с `MasterProfile` в LLM. Если оценка соответствия выше заданного порога, статус меняется на `MATCHED`.

### UC-3: Поиск контактов (Custom Email Discovery & Verification)
Для вакансий в статусе `MATCHED` система пытается найти контактное лицо и верифицирует его почту через собственный алгоритм (Permutation Engine -> MX Resolution -> Catch-All Check -> SMTP Ping).

### UC-4: Генерация пакета документов (LangGraph Pipeline)
Процесс создания документов не является линейным. ИИ-агент работает в рамках направленного графа:
1. Система оценивает скоринг.
2. Идет параллельная генерация Резюме и Сопроводительного письма.
3. Обязательный этап саморефлексии (Self-Correction): отдельный промпт-рецензент проверяет, не придумала ли модель несуществующий опыт.

### UC-5: Прямой отклик и Трекинг (Direct Outreach & Inbox Parsing)
Система автоматически отправляет сгенерированное письмо рекрутеру. Далее система слушает входящие письма (через Google Cloud Pub/Sub), парсит ответы с помощью ИИ и автоматически обновляет пайплайн отклика.
