# System Architecture & Infrastructure

## 1. Database Management
* **СУБД:** PostgreSQL (в Docker-контейнере).
* **ORM:** SQLAlchemy (асинхронный движок `asyncpg`).
* **Миграции:** Alembic. Все изменения структуры БД версионируются, скрипты миграций генерируются автоматически (`alembic revision --autogenerate`) и хранятся в репозитории.

## 2. AI Orchestration (LangGraph)
Пайплайн обработки вакансии реализуется как State Machine с использованием LangGraph.
* **State:** Состояние графа (Graph State) строго типизировано (Pydantic). Оно хранит текущую вакансию, профиль кандидата, извлеченные требования и черновики документов.
* **Nodes (Узлы):**
    1.  `Extract_Requirements`: Выделяет ключевые требования из сырого текста вакансии.
    2.  `Match_Profile`: Оценивает, насколько `MasterProfile` подходит под требования (возвращает score).
    3.  `Draft_Resume`: Формирует адаптированный JSON/Markdown резюме.
    4.  `Draft_CoverLetter`: Пишет персонализированное письмо.
    5.  `Reviewer`: Проверяет качество (наличие галлюцинаций, тон). В случае провала отправляет обратно на генерацию (циклическая связь).
* **Модель:** `gemini-3.5-flash` для всех текстовых операций внутри узлов.
