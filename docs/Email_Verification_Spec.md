# Component Specification: Email Discovery & Verification Engine

## 1. Назначение
Сервис для асинхронной генерации и валидации email-адресов рекрутеров с защитой от Catch-All доменов и обходом блокировок провайдеров через SOCKS5-прокси.

## 2. Основной Алгоритм (Pipeline)

1. **Permutation:** Генерация кандидатов (`first.last@domain.com`, `f.last@domain.com` и т.д.) на основе имени и домена.
2. **MX Resolution:** Асинхронный DNS-запрос для получения серверов домена с высшим приоритетом.
3. **Catch-All Check:** * Формирование заведомо несуществующего адреса (например, `random_uuid@domain.com`).
   * Выполнение SMTP Ping. Если ответ `250 OK`, домен помечается как `CATCH_ALL`. Проверка реальных адресов отменяется, генерируется событие для ручной обработки.
4. **SMTP Ping:** Если домен не Catch-All, выполняется пинг сгенерированных адресов до первого ответа `250 OK`. Соединение с MX-сервером осуществляется строго через SOCKS5-прокси.

## 3. Требования к реализации
* Строгая асинхронность (`asyncio`).
* Обработка сетевых ошибок и таймаутов при подключении к прокси и MX-серверам.
* Компонент не должен иметь жесткой привязки к БД (чистая бизнес-логика), принимая и возвращая Pydantic-модели.
